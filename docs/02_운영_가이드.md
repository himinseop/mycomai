# 02. 운영 가이드

## 사전 요구사항

- Docker Desktop (권장) 또는 Docker Engine + Compose v2
- `.env` 파일 설정 완료 (`.env.sample` 참고)

---

## 디렉토리 구조

```
mycomai/
├── db/                         # 데이터베이스 (git 제외)
│   ├── chroma_db/              # ChromaDB 벡터 DB
│   └── query_history.db        # SQLite (이력 + FTS5 인덱스)
├── data/                       # 수집된 JSONL 원본 (git 제외)
├── docker/                     # Docker 관련 파일
│   └── docker-compose.yml
├── src/company_llm_rag/        # 소스 코드
├── docs/                       # 운영 문서
├── .env                        # 실제 환경변수 (git 제외)
└── .env.sample                 # 환경변수 템플릿
```

## 주요 명령어

### 웹 서버 실행 (백그라운드)
```bash
docker-compose -f docker/docker-compose.yml up web -d
```
접속: http://localhost:8000

### 웹 서버 중지
```bash
docker-compose -f docker/docker-compose.yml stop web
```

### 데이터 수집 + 임베딩 + 적재 (전체 파이프라인)
```bash
docker-compose -f docker/docker-compose.yml up data-loader
```
- Jira, Confluence, SharePoint, Teams 순서로 수집 후 ChromaDB에 적재
- 콘텐츠 해시 비교로 변경분만 임베딩 생성 (비용 절감)
- 완료 후 컨테이너 자동 종료

### FTS5 인덱스 재구축
```bash
docker-compose -f docker/docker-compose.yml run --rm web python -m company_llm_rag.rebuild_fts
```
- data-loader 실행 후 또는 DB 복구 후 실행
- ChromaDB 전체 청크를 SQLite FTS5에 색인

### DB 문서 수 확인
```bash
docker-compose -f docker/docker-compose.yml run --rm web python -c \
  "from company_llm_rag.database import db_manager; print(db_manager.get_collection_stats())"
```

### 로컬 실행 (Docker 없이)
```bash
PYTHONPATH=src python3 -m company_llm_rag.rebuild_fts
PYTHONPATH=src python3 -c "from company_llm_rag.config import settings; print(settings.COLLECTION_NAME)"
```

## 수집 주기 관리

### 전체 재수집
```bash
# LOOKBACK_DAYS 미설정 시 전체 수집
docker-compose -f docker/docker-compose.yml up data-loader
```

### 증분 수집 (최근 N일)
`.env`에서 설정:
```
LOOKBACK_DAYS=7
```

### 크론 스케줄러 (자동 수집)
```bash
docker-compose -f docker/docker-compose.yml up cron-scheduler -d
```
스케줄은 `docker/crontab` 파일에서 편집.

## 문제 해결

### "database disk image is malformed" 오류
FTS5 인덱스 손상 시:
```bash
# 웹 서버 중지
docker-compose -f docker/docker-compose.yml stop web

# SQLite .recover로 이력 복구 후 FTS 재구축
# (자세한 절차는 history_store.py 주석 참고)

# FTS5 재구축
docker-compose -f docker/docker-compose.yml run --rm web python -m company_llm_rag.rebuild_fts

# 웹 서버 재기동
docker-compose -f docker/docker-compose.yml up web -d
```

### 웹 서버 응답 없음
```bash
docker-compose -f docker/docker-compose.yml restart web
```

### 임베딩 비용 과다
- `LOOKBACK_DAYS` 설정으로 증분 수집
- data_loader의 콘텐츠 해시 기반 스킵 로직이 정상 동작하는지 확인 (로그에서 `skipped` 카운트 확인)
