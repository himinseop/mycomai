# Issue #31 질의 개선 추진 계획

## 배경

- 대상 이력: `query_history` 77번
- 질문: `최근 등록된 지라 일감 보여줘`
- 문제 요약:
  - `최근`이라는 시간 조건이 답변에 명확히 반영되지 않음
  - 오래된 Jira 이슈가 함께 노출됨
  - 답변이 중간에 끊겨 완전성이 떨어짐
  - 최신성보다 유사도 중심으로 검색 결과가 상단에 배치됨

## 목표

- `최근`, `최신`, `방금 등록된`처럼 시간성이 강한 질의를 안정적으로 처리한다.
- Jira 질의에서 최신 등록/수정 이슈를 우선적으로 반환한다.
- 목록형 답변이 잘리지 않도록 응답 형식을 고정한다.
- 운영 중 동일 유형의 불만족 피드백을 줄인다.

## 개선 범위

- 검색 정렬 로직 개선
- 시간 의도 질의 감지 규칙 추가
- Jira 메타데이터 기반 기간 필터 적용
- 목록형 응답 템플릿 정비
- 운영 검증용 회귀 테스트셋 구축

## 세부 추진 과제

### 1. 시간 의도 질의 감지

- `최근`, `최신`, `최근 등록`, `최근 생성`, `요즘`, `방금 등록된` 등의 표현을 감지한다.
- 시간 의도 감지 시 일반 검색 모드가 아니라 `최신성 우선 모드`를 활성화한다.

### 2. Jira 시간 필터 정책 도입

- Jira 관련 질의에서는 `created`, `updated` 메타데이터를 기준으로 최근 30일을 우선 적용한다.
- 결과가 부족하면 최근 60일, 90일 순으로 점진 확대한다.
- 사용자가 기간을 명시한 경우 해당 기간을 우선 적용한다.

### 3. 최신성 가중치 정렬 추가

- 기존 유사도 기반 랭킹에 최신성 가중치를 추가한다.
- 오래된 이슈가 상위에 노출되는 문제를 줄이기 위해 recency boost를 반영한다.
- Jira 소스는 최신성 가중치를 일반 문서보다 강하게 적용한다.

### 4. 목록형 응답 템플릿 고정

- 답변 형식을 `최근 등록된 Jira 일감 N건` 형태로 통일한다.
- 각 항목에 다음 정보를 포함한다.
  - 이슈 키
  - 제목
  - 상태
  - 담당자
  - 생성일
- 길이 제한이 있으면 `상위 N건 기준`임을 명시하고 중간 절단 대신 명확히 요약한다.

### 5. 질의 해석 보강

- 사용자가 기간을 명시하지 않았을 경우 응답 첫 문장에서 기본 기준을 선언한다.
- 예시: `최근 30일 기준으로 등록된 Jira 일감을 보여드립니다.`

### 6. 회귀 테스트셋 구성

- 77번과 유사한 실패 질의를 10~20건 수집한다.
- `최신성 정확도`, `응답 완전성`, `사용자 기대 일치도`를 비교 지표로 삼는다.
- 개선 전/후 비교가 가능하도록 테스트 질의와 기대 결과 기준을 문서화한다.

## 추진 단계

### 1단계. 현황 점검

- Jira 문서 메타데이터에 `created`, `updated`, `status`, `assignee`가 안정적으로 적재되는지 확인
- 최신성 관련 실패 질의 샘플 수집

### 2단계. 설계

- 시간 의도 감지 규칙 정의
- 기본 기간 정책 정의

- 목록형 응답 템플릿 정의

### 3단계. 개발

- 검색 정렬 및 필터링 로직 수정
- Jira 질의 후처리 로직 추가
- 응답 포맷 정비

### 4단계. 검증

- 77번 질의 재현 테스트
- 회귀 테스트셋 기반 품질 비교
- 운영 화면에서 분석 결과 및 피드백 확인

### 5단계. 운영 반영
