# Issue #33 질의 개선 회귀 테스트셋

## 목적

- 상위 이슈 #31에서 정의한 최신성 질의 개선 요구사항을 코드와 문서 기준으로 재검증할 수 있게 한다.
- `최근 등록된 지라 일감 보여줘` 유형의 실패를 이후 변경에서 다시 만들지 않도록 자동 테스트와 수동 검증 기준을 함께 남긴다.

## 자동 검증 범위

- 최신성 질의 키워드 감지
- 사용자 지정 기간 파싱
- Jira 최근 기간 필터(자동 30/60/90일 확장)
- Jira 최근 기간 필터(사용자 지정 기간 우선)
- 최신성 점수 계산
- 프롬프트 내 기간 기준 선언
- Jira 목록 응답 포맷 지침 포함 여부

## 수동 검증 절차

1. 관리자 화면 또는 CLI에서 R1, R4, R5 질의를 각각 실행한다.
2. 답변 첫 문장에 실제 적용된 기간이 선언되는지 확인한다.
3. Jira 항목이 `이슈키 | 상태 | 담당자 | 생성일` 형식으로 나열되는지 확인한다.
4. 오래된 Jira 이슈가 최신 이슈보다 먼저 노출되지 않는지 확인한다.
5. 결과 수가 적을 때 자동 기간 확장 또는 `검색된 항목 기준` 문구가 자연스럽게 유지되는지 확인한다.

## 비교 지표

- 최신성 정확도: 상위 응답 Jira 항목이 실제 생성일 기준 내림차순에 가까운지
- 응답 완전성: 목록형 응답이 중간에 끊기지 않고 마무리되는지
- 기대 일치도: 기간 선언과 실제 필터 기간이 일치하는지

## 비고

- 현재 자동 테스트는 LLM 최종 자연어 생성 자체를 검증하지 않는다.
- 대신 프롬프트 지침, 기간 해석, 정렬/필터 핵심 로직을 고정해 동일 유형 회귀를 조기에 탐지한다.
