# 01. 시스템 소개

## 개요

Mycomai는 회사 내부 데이터를 수집·색인하여 자연어 질의응답을 제공하는 RAG(Retrieval-Augmented Generation) 시스템입니다.

Jira, Confluence, SharePoint, Teams 데이터를 ChromaDB 벡터 DB에 저장하고, OpenAI GPT 모델을 통해 검색 기반 답변을 생성합니다.

---

## 아키텍처

```
[데이터 소스]                  [수집/적재]             [서비스]
Jira ──────────────────┐
Confluence ────────────┤──▶ data_extraction  ──▶ data_loader.py
SharePoint ────────────┤       (JSONL)             │
Teams (채널/채팅) ──────┘                           ▼
                                             ChromaDB (벡터 DB)
                                             SQLite (FTS5 역인덱스 + 이력)
                                                    │
                                                    ▼
                                         retrieval_module.py
                                          (벡터 + FTS 하이브리드 검색)
                                                    │
                                                    ▼
                                            rag_system.py
                                          (OpenAI GPT 답변 생성)
                                                    │
                                                    ▼
                                          web_app.py (FastAPI)
                                             :8000
```

---

## 주요 구성 요소

| 모듈 | 역할 |
|------|------|
| `config.py` | 환경변수 기반 전역 설정 (싱글톤) |
| `database.py` | ChromaDB 연결 관리 (Lazy init) |
| `data_loader.py` | JSONL → ChromaDB 적재. 콘텐츠 해시 비교로 변경분만 임베딩 |
| `retrieval_module.py` | 벡터 검색 + FTS5 키워드 검색 하이브리드 (RRF 재랭킹) |
| `rag_system.py` | 검색 + LLM 조합 RAG 파이프라인 |
| `history_store.py` | 질의 이력 저장 (SQLite WAL), FTS5 인덱스 관리 |
| `web_app.py` | FastAPI 웹 서버 (채팅 UI + 어드민 대시보드) |
| `no_answer_analyzer.py` | 답변 불충분 케이스 자동 분석 |
| `rebuild_fts.py` | FTS5 인덱스 초기 구축 / 재구축 유틸리티 |

## 데이터 소스별 수집 방식

| 소스 | API | 페이지네이션 |
|------|-----|-------------|
| Jira | REST API v3 (`/search/jql`) | `nextPageToken` + `isLast` |
| Confluence | REST API (`/rest/api/content`) | `size < limit` 체크 |
| SharePoint | Microsoft Graph API | `@odata.nextLink` |
| Teams 채널 | Microsoft Graph API | `@odata.nextLink` |
| Teams 채팅 | Microsoft Graph API (`/chats/{id}/messages`) | `@odata.nextLink` |

---

## 검색 방식

하이브리드 검색 후 RRF(Reciprocal Rank Fusion)로 최종 순위를 결정합니다.

1. **벡터 검색**: ChromaDB cosine similarity (text-embedding-3-small)
2. **FTS5 키워드 검색**: SQLite doc_fts 역인덱스 (BM25 기반 prefix 검색)
3. **소스 부스트**: 소스별 가중치 적용 (Jira > Teams > Confluence > SharePoint)
4. **직접 조회**: 쿼리에 Jira 이슈 키(`PROJECT-123`) 포함 시 해당 청크 우선 삽입

## 데이터베이스 구조

```
db/
├── chroma_db/          # ChromaDB 벡터 저장소 (~179,979개 청크)
└── query_history.db    # SQLite
    ├── query_history   # 질의 이력 (세션, 질문, 답변, 참고문서, 피드백)
    ├── doc_fts         # FTS5 가상 테이블 (키워드 역인덱스)
    └── app_settings    # 어드민 설정 (KV 스토어)
```
