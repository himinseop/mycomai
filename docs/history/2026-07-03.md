- **원인**: 로우데이터/회원리스트/매장목록/취소이력 등 거대 엑셀이 검색 상위 오염("포인트 적립"→`2019로우데이터.xlsx` top-1). #48 콘텐츠 상한 이전 통째 인덱싱된 orphan 청크 대량 잔존.

- **조치**: (1) loader가 스프레드시트 콘텐츠 >1MB면 인덱싱 스킵(`LOADER_SKIP_SPREADSHEET_OVER_KB`, 재수집 방지), (2) 1회 정리 스크립트 `scripts/cleanup_spreadsheet_junk.py`로 ChromaDB+FTS 삭제. 삭제 전 `db/chroma_db.bak-20260703`, `db/search_index.db.bak-20260703` 백업.
- **검증**: "포인트 적립" top 결과가 로우데이터 → 실제 관련 문서로 개선. PII(회원/이메일 리스트)도 제거됨.
- **⚠️ 미회수**: ChromaDB soft-delete라 디스크(6.5GB)·메모리는 즉시 감소 안 함. 실제 절감은 컬렉션 **컴팩션/재구축** 필요(후속).

## UX: 질문 해석·검색 근거를 답변과 하나의 흐름으로 노출 (#52 활성화)
- **해석 문장 → 찾은 문서 → 실제 답변**이 하나의 답변 버블에서 이어짐 (별도 박스 X, 사라지지 않고 유지). 이벤트: `interpretation` → `sources` → `token…`.
- 해석 문구는 **LLM이 다듬은 자연어 문장**(`understanding` 필드): "…질문이시군요. 관련 자료를 찾아볼게요." (검색용 `rewritten`과 별도, 같은 재작성 LLM 호출에서 함께 생성).
- `QUERY_REWRITE_ENABLED` **기본 on 전환** — 해석이 검색에도 반영. 매 질의 +재작성 LLM 호출(수백ms)이나, 해석을 즉시 보여줘 체감 지연은 오히려 완화.
- 프론트: 답변 콘텐츠(`.qa-a-content`) 앞에 `.ans-lead`(해석)·`.ans-found`(근거) preamble로 렌더, `finalizeTurnEl`까지 유지.

### 참고문서 vs 검색결과 차이 + 저장·표시 개선 (sess-v810jiv 조사)
- **원인**: 검색결과(retrieved_docs)=RRF+rerank 상위 청크(중복 문서 포함, 예 10), 참고문서(references)=`_build_references`가 URL 중복제거 + 거리(0.35) 필터 + 인용 우선으로 축소(예 4). 의도된 축소이나 "문서 N건" 표시가 검색결과 수라 혼란.
- **개선**: (1) 근거 문구를 **실제 참고문서 수(ref_count)**로 확정(meta 이벤트), 검색 진행 중엔 "…답변하고 있어요". (2) 근거 문구를 **답변과 동일 텍스트 스타일**로(연속된 답변 느낌). (3) 질문해석(`understanding`)을 perf_json에 저장 → **관리자 '질문 상세'에 해석 + "문서 N건을 참고했어요"를 답변의 일부로 표시**.
