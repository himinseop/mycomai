## 거대 스프레드시트 raw-data 검색 오염 제거 (#55)
- **원인**: 로우데이터/회원리스트/매장목록/취소이력 등 거대 엑셀이 검색 상위 오염("포인트 적립"→`2019로우데이터.xlsx` top-1). #48 콘텐츠 상한 이전 통째 인덱싱된 orphan 청크 대량 잔존.
- **범위**: 스프레드시트+청크>1000 → **12개 문서 247,191 청크 삭제** (409,522 → 162,331, 60% 감축). 정상 문서(PDF/docx 가이드, `_TC_`)는 영향 없음.
- **조치**: (1) loader가 스프레드시트 콘텐츠 >1MB면 인덱싱 스킵(`LOADER_SKIP_SPREADSHEET_OVER_KB`, 재수집 방지), (2) 1회 정리 스크립트 `scripts/cleanup_spreadsheet_junk.py`로 ChromaDB+FTS 삭제. 삭제 전 `db/chroma_db.bak-20260703`, `db/search_index.db.bak-20260703` 백업.
- **검증**: "포인트 적립" top 결과가 로우데이터 → 실제 관련 문서로 개선. PII(회원/이메일 리스트)도 제거됨.
- **⚠️ 미회수**: ChromaDB soft-delete라 디스크(6.5GB)·메모리는 즉시 감소 안 함. 실제 절감은 컬렉션 **컴팩션/재구축** 필요(후속).
