## 도메인별 LLM 인사이트 API Phase 1~3 (#56, feature/56-insight-api)
- **구조**: `insight_api/` 패키지 — 도메인 레지스트리(`POST /api/v1/insights/{domain}`). 도메인 = 입력 스키마 + 통계 선계산 + 프롬프트 + 응답 파싱. **수치는 서버가 계산, LLM은 해석만**.
- **Phase 1**: X-API-Key 인증(SHA-256 해시 저장) + 도메인 scope + IP allowlist(CIDR), `sales` 도메인(총액/추이/차원별/전기대비/z-score 이상치), `api_call_history` 이력(원본 데이터 미저장), 키 발급 CLI(`scripts/api_key_issue.py`).
- **Phase 2**: 관리자 API 탭(키 발급 1회 노출 모달/차단/재활성, 호출 이력 필터 조회), 키별 분당 rate limit(sliding window, 429도 이력 기록).
- **Phase 3**: `voc` 도메인(평점 분포/부정 비율/카테고리 집계/건수 급증 감지 + 부정 우선 샘플로 LLM theme 도출)으로 **레지스트리 확장 구조 검증** — 파일 1개+등록+프롬프트만으로 도메인 추가, 공통 파이프라인 무변경. 샘플 원문은 응답/이력 비노출.
- **E2E에서 발견·수정**: gpt-4o-mini가 한글 금액 단위 변환 오류(1,605만→"1억 6050만") → 금액·증감률 표시 문자열(`*_display`)을 서버가 확정하고 프롬프트에서 그대로 쓰도록 강제. 재검증에서 수치 전부 정확.
- **검증**: 테스트 23건(설계 TC 1~12 + rate limit + voc) 통과, 실 LLM E2E(매출·VOC) 수기 검산 일치.

- **LLM 모델 스위치**: 설정 탭에서 역할별(답변 생성/질문 재작성/요약·보조/인사이트 API) 모델을 셀렉트로 변경. `llm/factory.current_model(role)` — app_settings 오버라이드 > .env 기본값, 재시작 없이 즉시 반영. 화이트리스트 검증(422). ollama provider에서는 비활성.
- **API 메뉴 → 설정 탭 통합**: 별도 API 탭 제거, 설정 탭 안에 API 클라이언트·호출 이력 카드로 이동.
- **호출 이력 표시 깨짐 수정**: 원본 JSON 노출 → 읽기 좋은 요약(기간·행수·차원·선택방법·질문)으로 변환, UTC → 로컬 시각(fmtDate), 오류는 별도 줄, 테이블 가로 스크롤 래퍼.
- **접속 이력**: 채팅방(/)·어드민(/admin) 진입 시 IP·시각·User-Agent 기록(web_access_log, 30일 보관) → 대시보드 하단 카드로 조회. ⚠ Docker Desktop NAT 특성상 소스 IP가 192.168.65.1로 보임 — 실 IP는 앞단 프록시(XFF) 필요.
